# AGENTS.md

## Project Context

This project is a production-grade monitoring SaaS inspired by UptimeRobot.

Stack:

* Frontend: React + TypeScript + Vite
* Backend: NestJS + Prisma
* Database: PostgreSQL
* Mobile: Flutter

The UI must feel:

* modern
* premium
* iOS-inspired
* clean
* professional
* consistent across pages

Avoid generic AI-looking dashboards.

---

## Frontend Design Rules

Always apply the `frontend-design` skill when:

* creating pages
* editing UI
* redesigning components
* improving layouts
* adding charts/tables/forms
* creating dashboards
* improving spacing/typography/colors

### Visual Direction

Use:

* layered surfaces
* soft borders
* subtle gradients
* refined spacing
* premium typography hierarchy
* elegant shadows
* smooth hover states
* modern table designs
* minimal but high-end visuals

Avoid:

* generic Tailwind dashboard look
* overly saturated gradients
* purple-on-white AI aesthetic
* oversized shadows
* cramped layouts
* inconsistent spacing

---

## Code Rules

* Never rewrite unrelated files.
* Preserve existing business logic.
* Reuse shared styles/components when possible.
* Keep components modular.
* Keep TypeScript strict-safe.
* Avoid breaking API contracts.

---

## UX Rules

* Prioritize visual hierarchy.
* Maintain consistent button heights.
* Preserve responsive behavior.
* Improve empty states/loading states.
* Use smooth transitions only where valuable.

---

## Output Expectations

When modifying UI:

1. Explain briefly what will change.
2. Modify only required files.
3. Keep diffs minimal.
4. Maintain production quality.

When uncertain:

* ask before major architectural changes
* never invent backend contracts
